What is this report?

This view is not stored in the database. It is generated by combining data from multiple tables.

  • Billing — who the customer is
  • Menu — what is the bill
  • Enrollment — Which customer has what amount of bill

The system dynamically joins these tables using SQL JOIN operations to produce a meaningful report.

Data flow: billings → enrollments → menus

SQL used to generate this report (JOIN)

Read-only 
SELECT
    e.id AS enrollment_id,
    b.name AS name,
    m.title AS menu_title,
    e.enrolled_on
FROM enrollments e
JOIN menus s ON e.billing_id = b.id
JOIN billings c ON e.menu_id = m.id;
Note: menus information is not duplicated in the billings table. This follows database normalization principles.
